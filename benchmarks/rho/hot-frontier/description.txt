hot-frontier

This benchmark is one rho process family: N receives and N sends all compete
on the same channel, so the strict-core rho frontier has N*N one-step
successors. It is the sharp benchmark for full COMM-frontier enumeration.

Files:
- sizes.tsv: default sizes for this benchmark only.
- generate.sh SIZE OUT_DIR: writes hot_frontier_SIZE.metta,
  hot_frontier_SIZE.mrho, and hot_frontier_SIZE.rho.
- run.sh: runs only hot-frontier against CeTTa lib_rho, CeTTa --lang rhocalc,
  and f1r3node rholang-cli.

Semantic caveat: CeTTa rho lanes compute the strict-core one-step successor
frontier. rholang-cli evaluates Rholang to quiescence.

CeTTa lib_rho prints the full rho:steps reduction frontier; rholang-cli prints a deploy-cost summary of roughly 70 bytes. The measured wall time therefore includes asymmetric output-formatting cost. Treat these rows as engine-end-to-end measurements, not scheduler-to-scheduler measurements.
