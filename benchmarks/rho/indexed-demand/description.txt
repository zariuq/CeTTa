indexed-demand

This benchmark is the policy-aware counterpart to demand-index. The source is
authored as MeTTa tables containing all watchers and all facts across four
topics. Each row carries both a symbolic topic tag for MeTTa filtering and a rho
name for communication. This is a semantic split, not syntax ambiguity: bare
topic symbols are policy data, while `$topic...` atoms are rho name identities.
MeTTa filters those tables to one selected topic, synthesizes the rho network,
and calls rho.step.

The point is algorithmic: --lang rhocalc can run the already-synthesized
process, but it cannot consume the watcher/fact tables, choose a topic index,
and build only the relevant rho network. The Rholang lane receives the
equivalent filtered network for cross-engine timing.

Files:
- sizes.tsv: table sizes for this benchmark only.
- generate.sh SIZE OUT_DIR: writes indexed_demand_SIZE.metta and
  indexed_demand_SIZE.rho.
- run.sh: runs only indexed-demand against CeTTa lib_rho and f1r3node
  rholang-cli.

Use TARGET_TOPIC=0..3 to select the indexed topic. The default is 0.

CeTTa lib_rho prints the full rho:steps reduction frontier; rholang-cli prints a deploy-cost summary of roughly 70 bytes. The measured wall time therefore includes asymmetric output-formatting cost. Treat these rows as engine-end-to-end measurements, not scheduler-to-scheduler measurements.
