fanout

This benchmark is one rho process family: a single broadcast receive fans one
message out to N sink channels. The `.mrho`/`.rho` lanes are real strict-core
rho programs that `--lang rhocalc` runs to quiescence. The `.metta` lane is a
builder/demo that emits the same rho process as data using small helper rules.

Files:
- sizes.tsv: default sizes for this benchmark only.
- generate.sh SIZE OUT_DIR: writes fanout_SIZE.metta, fanout_SIZE.mrho, and
  fanout_SIZE.rho.
- run.sh: runs only fanout against CeTTa --lang rhocalc runtime evaluation
  and f1r3node rholang-cli.

The `.mrho` file is CeTTa's strict-core rho concrete syntax; the `.rho` file
is the surface rho/Rholang spelling used for cross-engine runs.

CeTTa prints the residual process after deterministic reduction;
rholang-cli prints a deploy-cost summary of roughly 70 bytes. The measured
wall time therefore includes asymmetric output-formatting cost. Treat these
rows as engine-end-to-end measurements, not scheduler-to-scheduler
measurements.
