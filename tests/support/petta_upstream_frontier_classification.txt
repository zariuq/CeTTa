# Native --lang petta upstream frontier classification
#
# Scope:
# - 89 upstream examples are currently in petta_upstream_reasonable_examples.txt.
# - 43 upstream examples are currently in petta_upstream_current_gaps.txt.
# - 31 upstream examples are deferred from both lists.
#
# The 74 frontier examples below are not a single denominator of semantic
# defects. They are grouped by the capability layer most likely to retire them.
# "deferred" means "not in the every-run broad lane yet", not "out of scope".
# The current 47-file gap queue is also classified in:
#   tests/support/petta_upstream_gap_root_causes.txt

P0 import-module parity
-----------------------
Needed because many higher examples fail before their real semantics run.

Current root needs:
- PeTTa-style relative imports from files run by absolute path.
- `(library name)` resolution against PeTTa libraries.
- extensionless local imports such as `!(import! &self fibsmart)`.
- recursive import metadata for callable syntax and relation lowering.

Files:
- builin_types.metta
- fibsmartimport.metta [deferred]
- patrick_iterate_fib.metta
- patrick_iterate_quad.metta
- patrick_test.metta
- spaces_find.metta
- spaces_removeallatoms.metta
- spaces_succeedspredicate.metta
- tilepuzzle.metta [deferred]

P0 primitive surface coverage for lib_he.metta
----------------------------------------------
Needed because CeTTa `--lang he` already supports much of this surface, and
`--lang petta` should not lose those core host-language conveniences.

Current root needs:
- Load upstream `lib_he.metta` as ordinary source under PeTTa import rules.
- remaining evaluation-control surfaces and long-running recursive control.
- HE-style `id`, `pow-math`, `for-each-in-atom`, and remaining recursive helpers.
- remaining match-template binding through collapse and recursive control.
- recursive `cons-atom` / `decons-atom` / `size-atom` behavior.

Files:
- he_minimalmetta.metta
- test_collapse_size_bug.metta
- test_process_by_tag.metta [deferred]
- test_recursive_size_atom.metta

P0 core value and operator parity

Needed because these are ordinary PeTTa language behaviors, not advanced

P0 match, binding, and atomspace mutation
-----------------------------------------
Needed because this is the same semantic layer as the already-fixed `once`,
imported syntax args, and equation hygiene bugs.

Current root needs:
- nested `match` bodies preserving bindings through side effects.
- comma/conjunction match patterns.
- nested committed-choice behavior beyond the current direct `cut` and
  `matchsingle` probes.
- `remove-all-atoms`, state, transactions, and dynamic specialization after
  mutation.
- binding destructuring through nested `let` and `let*`.

Files:
- letext.metta
- letlet.metta
- matchnested.metta
- matchnested2.metta
- matchtypes.metta
- mutex_and_transaction.metta
- state.metta

---------------------------------

substrate because many failures are likely secondary.

Current root needs:

- parametric and dependent type matching.
- function specialization metadata from typed callable heads.

Immediate foundation-up work order
----------------------------------
1. Import-module parity: keep file-path/library/extensionless imports
   integrated with PeTTa library roots so tilepuzzle, Roman, PLN/NARS, and
   spaces examples can exercise their real semantics.
2. Primitive surface coverage: make upstream `lib_he.metta` load verbatim
   under `--lang petta`, including catch/error, unquote/eval, HE math/atom
   utilities, and match-template binding through collapse.
3. Core operators: atomops, math, parse/string comments, quote/eval/call/
   reduce, and higher-order intrinsic operators not covered by callable values.
4. Match/mutation: nested match with side effects, `remove-all-atoms`, state,
   transactions, and dynamic callable inventory updates.
5. Types and specialization: get-type, dependent/nondet type dispatch,
   specialization metadata.
6. Search/streams/tabling/hyperpose: only after P0 role/import/match layers are
   trustworthy.
7. PLN/NARS named lane: after imports, HE, math, and query/search basics.
8. Optional interop lanes: Python/torch, Prolog/Git, and API-held LLM examples.
